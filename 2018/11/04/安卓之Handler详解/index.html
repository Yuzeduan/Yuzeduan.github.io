Handler异步信息处理机制

Handler机制介绍

Handler机制是android系统提供给我们的一种消息处理机制，最常见的应用就是在子线程更新UI操作,其中Activity中所有的生命周期方法的回调都是应用的Handler机制。

Handler机制主要是为了解决安卓系统中多线程并发的问题，试想如果如果在没有加锁的情况下在多个子线程中更新主线程即UI线程界面，会造成界面错乱，我们也无法控制界面显示，如果在UI线程中加锁的话又会造成性能下降，影响用户体验

Handler机制的四大部分组成

1.Message

massage是在线程之间传递的信息,它可以在内部携带少量的信息,用于在不同线程之间交换数据,一个message对象包含一个自身的描述信息和一个可以发给handler的任意数据对象。这个对象包含了两个int类型arg1和arg2的字段和一个object类型的obj字段.实例化Message最好的方法是调用Message.obtain()或Handler.obtainMessage()（实际上最终调用的仍然是Message.obtain()），因为这两个方法都是从一个可回收利用的对象池中获取Message的.

2.Handler

常见构造方法:
Handler() 默认构造方法，与当前线程及其Looper实例绑定。如在主线程中执行new Handler()，那么该handler实例所绑定的便是 UI 线程和 UI 线程绑定的Looper实例。

Handler(Handler.Callback callback) 与当前线程及其Looper实例绑定，同时调用一个callback接口

Handler(Looper looper) 将该新建的handler实例与指定的looper对象绑定

Handler(Looper looper, Handler.Callback callback)
指定该handler实例所绑定的looper实例并使用给定的回调接口进行消息处理。

它主要用于发送和处理信息的,发送信息可以是sendMessage()方法(常用),也可以通过post()方法将一个Runnable对象投递到Handle中的Looper去处理.

注意:Handler创建时候会采用当前线程的Looper来构建内部的消息循环系统,如果当前线程没有Looper,那么会报错

3.MessageQueue

MessagQueue是消息队列,主要用于存放所有通过Handler发送的信息,通过一个单链表的数据结构来维护消息列表,每个线程中只会有一个MeaasgeQueue对象,对外提供插入和删除的工作.

4.Looper

Looper是线程用来运行消息循环(message loop)的类,调用了Looper的Loop()方法之后,就会进入一个无限循环中,然后每当发现MessageQueue中存在一条消息,就会将它取出,每个线程中只有一个Looper对象.

ThreadLocal的工作原理

1.简介

ThreadLocal是一个线程内部的数据存储类,通过它可以在指定的线程中存储数据,数据存储以后,只有在指定的线程中可以获得存储的数据,对于其他线程则无法获取到数据.

2.使用场景

当某些数据是以线程为作用域并且不同线程具有不同的数据副本的时候,就会考虑使用ThreadLocal,对于Handler来说,Looper的作用域就是线程并且不同线程具有不同的Looper,此时可通过ThreadLocal实现Looper在线程的存取.

3.使用机理

ThreadLocal常用的就是set和get方法.

  • 使用Looper.myLooper()获取该线程的Looper对象

  • 使用Looper.prepare()方法新建一个Looper对象,个线程中只能创建一个Looper对象，如果调用过Looper.prepare()后再次调用就会抛出异常

  • set()方法的实现机理

在Thread类的内容有一个成员专门用于存储线程的ThreadLocal的数据–ThreadLocal.Values localValues，因此获取当前线程的ThreadLocal数据就变得异常简单了。如果localValues的值为null，那么就需要对其进行初始化，初始化后再将ThreadLocal的值进行存储。在localValues内部有一个数组：private Object[] table，ThreadLocal的值就是存在在这个table数组中.
put方法将ThreadLocal的值存储到table数组中的,其存储规则是ThreadLocal的值在table数组中的存储位置总是为ThreadLocal的reference字段所标识的对象的下一个位置.

  • set()方法的实现机理

get()方法取出当前线程的localValues对象，如果这个对象为null那么就返回初始值，初始值由ThreadLocal的initialValue方法来描述，默认情况下为null.
如果localValues对象不为null，那就取出它的table数组并找出ThreadLocal的reference对象在table数组中的位置，然后table数组中的下一个位置所存储的数据就是ThreadLocal的值.

消息队列的工作原理

MessageQueque 主要包含两个操作,插入和读取,读取操作本身会伴随着删除操作,插入和读取的对应的方法分别为enqueueMessage和next.通过单链表结构来维护消息列表,单链表在插入和删除上比较有优势.

  • enqueMessage方法的实现机理

最后调用的queue.enqueMessage(msg, uptimeMillis),实现的是单链表的插入操作.

  • next方法实现机理
    next方法是一个无限循环的方法,如果消息队列没有消息,那么next方法会一直堵塞在那里,当有新消息到来的时,next方法会返回这条信息并将其从单链表中移除.
    (如果messageQueue有消息，自然是继续取消息；如果已经没有消息了，此时该线程便会阻塞在该next()方法的nativePollOnce()方法中,线程便会释放CPU资源进入休眠状态)

Looper的工作原理

Looper扮演着消息循环的角色,它会不停地从MessageQueue中查看是否有新消息,如果有新消息就会立刻处理,否则就会一直阻塞在那里.

  • Looper的构造方法

  • 创建Looper

    Looper可以通过prepare方法还有prepareMainLooper方法(主要是给主线程ActivityThread创建Looper,本质也是prepare实现).

  • Looper的退出

    Looper的quit和quitSafely来退出一个looper,前者会直接退出Looper,而quitSafely是设定一个退出标记,只有等消息处理完毕后才退出.
    在子线程中,如果手动为其创建Looper对象,在所有事情处理完毕后,要调用quit方法终止消息循环,否则子线程会一直处于等待状态,如果退出Looper后,这个线程就会终止.

  • Looper的loop方法

loop方法通过MessageQueue的next方法取消息,当next方法返回null时跳出循环,否则会无限循环下去,调用了looper的quit方法,就会调用MessageQueue的quit和quitSafely,从而使next方法返回null,结束looper的循环.因此,Looper必须退出.
