安卓之View的学习一

发表于 2018-11-04 | 分类于 Android
字数统计: 1k | 阅读时长 ≈ 3

1.view的基础知识

1.1 View的定义

View是Android中所有控件,布局的基类,其中Viewgroup是控件组,即一组View,ViewGroup也是继承自View.

1.2 Veiw的位置参数

  • View的位置是由他的四个顶点来确定的,其中top是指左上角的纵坐标,left是左上角的横坐标,right是右下角的横坐标,bottom是右下角的纵坐标.这些坐标都是相对于View的父容器来说的,是相对坐标,x轴和y轴正方向是为右和下.

  • View的宽高和坐标的关系

  • 获取四个参数(提供get/set方法)

  • View在3.0后加入了额外的几个参数,其中x和y是View左上角的坐标,translationX和translationY是View左上角相对于父容器的偏移量,这些参数也是相对于父容器的坐标,其中偏移量默认值为0,View同样给这几个参数提供get/set方法.

注意 View在平移的过程中,top和left是原始的值,不会改变.

1.3 MotionEvent和TouchSlop

1.3.1 MotionEvent

  • 典型的触发事件类型
    • ACTION_DOWN : 手指刚触碰屏幕
    • ACTION_MOVE : 手指在屏幕上移动
    • ACTION_UP :手指从屏幕上松开的一瞬间
  • 通过MotionEvent对象获取点击事件发生的x和y坐标
    • getX/getY : 返回的是相对View相对于当前View左上角的x和y坐标
    • getRawX/getRawY :返回的是相对于手机屏幕左上角的ｘ和y坐标.

1.3.2 TouchSlop

它是系统所能识别出的被认为是滑动的最小距离,它是个常量,在不同的设备这个值可能不同,可通过ViewConfiguration.get(getContext()).getScaledTouchSlop().

1.4 VelocityTracker,GestureDetector和Scoller

1.4.1 VelocityTracker

速度追踪,用于追踪手指在滑动过程中的速度,包括水平和竖直方向的速度,首先在View的onTouchEvent方法中追踪当前单击事件的速度

获取当前速度

在传入的时间段中所滑动的的像素数就是速度,手指如果逆坐标系正方向滑动的话,所产生的速度就是个负值.
不需要使用它的时候

1.4.2 GestreDetector

手势检测,用于辅助检测用户的单击,滑动,长按,双击等行为.

  • 内部的回调接口
  1. OnGestureListener，这个Listener监听一些手势，如单击、滑动、长按等操作： 2. OnDoubleTapListener，这个Listener监听双击和单击事件。
  2. ContextClickListener，ContextClick就是鼠标右键
  3. SimpleOnGestureListener，实现了上面三个接口的类，拥有上面三个的所有回调方法,一般选择这个,不用重写所有的方法
  • Android中手势交互的执行顺序
    1. 手指触碰屏幕时，触发MotionEvent事件！
    2. 该事件被OnTouchListener监听，可在它的onTouch()方法中获得该MotionEvent对象！
    3. 通过GestureDetector转发MotionEvent对象给OnGestureListener
    4. 我们可以通过OnGestureListener获得该对象，然后获取相关信息，以及做相关处理！
  • 实现手势检测的步骤
    1. 创建GestureDetector对象，创建时需实现GestureListener传入
    2. 将Activity或者特定组件上的TouchEvent的事件交给GestureDetector处理即可

实现OnGestureListener需要实现所有的手势，可能我针对的仅仅是滑动，但是你还是要去重载,官方另外给我们提供了一个SimpleOnGestureListener类 只需把上述的OnGestureListener替换成SimpleOnGestureListener即可.
所有的回调方法的返回值都是boolean类型，和View的事件传递机制一样，返回true表示消耗了事件，flase表示没有消耗。

1.4.3 Scoller

弹性滑动对象,用于实现View的弹性滑动.
