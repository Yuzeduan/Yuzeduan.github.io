Java注解入门（Annotation）

1.何为元数据？

学习Java注解，就要先了解下何为元数据

  • 元数据是关于数据的数据，元数据是添加到方法、类等上的额外信息，对数据进行说明描述的数据。
  • 元数据的作用
    • 编写文档：通过代码中的元数据生成JavaDoc文档
    • 代码分析：通过代码里标识的元数据对代码进行分析
    • 编译检查：通过代码里标识的元数据让编译器执行检查
  • 通过提供为程序元素（类、方法等）附加额外数据的标准方法，元数据功能具有简化和改进许多应用程序开发领域的潜在能力，其中包括配置管理、框架实现和代码生成。

2.何为注解？

  • 注解是JDK1.5之后引入的一个新特性，@interface类型便是注解了，它就是Java平台的元数据。
  • 注解机制的出现，使我们可以在代码中添加自定义注释，可以通过反射获取注解的属性，其作为程序的元数据嵌入程序中。
  • 注解本身除了定义信息外其实什么都没做，它和业务逻辑没有关系，其不影响代码的执行，如果要使注解起一定的作用，只有通过解析工具或编译工具对Annotation中的信息进行解析和处理。
  • 例如：当我们使用@Override却没有重写父类的方法，编译器会报错，当我们使用ButterKnife这种工具时会有类文件被自动生成…这些都不是注解负责的，它们主要交由解析工具或者是编译工具等消费器来处理。
  • 注解中的数据可以在编译、运行时进行解析和使用，也可以单纯为编码人员提供很多的信息。

3.何为元注解?

元注解是描述其他注解的注解，其本身也由元注解描述

3.1 @Documented

@Documented修饰的Annotation类将会被javadoc工具提取成文档，即所有使用该Annotation修饰的程序元素的API文档中将会包含该Annotation说明。

3.2 @Inherited

@Inherited指定被它修饰的Annotation将具有继承性，即如果某个类使用了@Xxx注解（定义该Annotation时使用了@Inherited修饰）修饰，则其子类将自动被@Xxx修饰。

3.3 @Retention

  • @Retention表示该注解类型的注解保留的时长，特殊地，当注解类型声明中没有@Retention元注解，则默认保留策略为RetentionPolicy.CLASS。
  • 保留策略（枚举类型）
    • SOURCE：仅存在源文件中，经过编译之后，便丢弃该注解。
    • CLASS：存在源文件和编译生成的字节码文件，但在运行时被JVM忽略。
    • RUNTIME：保存在运行时JVM中，可通过反射性获取。

3.4 @Target

  • 表示该注解类型的所适用的程序元素类型。特殊地，当注解类型声明中没有@Target元注解，则默认为可适用所有的程序元素。
  • 程序元素（ElementType，枚举类型）
    • ElementType.TYPE:只能描述类、接口或enum
    • ElementType.FIELD:只能描述实例变量
    • ElementType.METHOD:只能描述方法
    • ElementType.PARAMETER:只能描述方法的参数
    • ElementType.CONSTRUCTOR:只能描述构造器
    • ElementType.LOCAL_VARIABLE:只能描述局部变量
    • ElementType.ANNOTATION_TYPE:只能描述另一个注释
    • ElementType.PACKAGE: 只能描述包名

3.5 @Repeatable（Java 8）

  • jdk1.8新增的该注解用于指示被该注解修饰的注解可以在用一个元素中多次使用

  • 对比Java1.8前和之后使用重复注解的解决方案

  • 另一个注解来存储重复注解，在使用时候，用存储注解来扩展重复注解，重复注解加上@Repeatable修饰，指向存储注解。

4.何为内建注解

即Java本身提供的多种注解，其主要实现了元数据的第二个作用：编译检查。

常见的内建注解

4.1 @Override

  • 用于告诉编译器，当前方法要重写父类的该方法。如果某个方法带有该注解但并没有覆写超类相应的方法，则编译器会生成一条错误信息。如果父类没有这个要覆写的方法，则编译器也会生成一条错误信息。
  • 可适用的元素是方法，且仅仅是保留在源文件内

4.2 @Deprecated

  • 用于告知编译器，某一程序元素(比如方法，成员变量)不建议使用了（即过时了）。
  • @Deprecated可适合用于除注解类型声明之外的所有元素，保留时长为运行时。
  • 使用的时候最好是同时使用Java Doc的@deprecated符号，用来描述当前类、方法或者字段是不赞成使用的，并且告诉开发者应该用哪个对象替换，进行注释。

4.3 @SuppressWarnings

  • 用于告知编译器忽略特定的警告信息，当使用该注解后，则不会发出警告。
  • 可适合用于除注解类型声明和包名之外的所有元素，保留在字节码文件中。
  • 注解有方法value(）,可支持多个字符串参数，用户指定忽略哪种警告。
  • 参数（部分常用的）
    • deprecation：使用了过时的类或者方法的警告
    • unchecked：执行了未检查的转换时的警告
    • fallthrough：当switch程序块进入下一个case时候没有break的警告
    • serial：忽略在serializable类中没有声明serialVersionUID变量
    • unused：抑制没被使用过的代码的警告
    • finally：抑制finally模块没有返回的警告
    • rawtypes：使用泛型时忽略没有指定相应的类型
    • all：抑制所有警告

4.4 @FunctionalInterface(jdk1.8)

  • 告知编译器，检查这个接口，保证该接口是函数式接口，即只能包含一个抽象方法，否则就会编译出错。

5.创建自定义注解

学习完元注解和java的内建注解，我们可以自己定义一个注解

  • 定义成员变量的规则
    • 其定义是以无形参的方法形式来声明的，即注解方法中不带任何参数
    • 注解方法返回值类型：基本类型、String、Enums、Annotation以及前面这些类型的数组类型
    • 注解方法可有默认值，比如default “hello”，默认website=”hello”

  • 注意点

    • 注解中也可以没有成员变量，因此解析这类注解进行操作时，仅以是否包含该注解来进行操作
    • 当注解中有成员变量时，若没有默认值，需要在使用注解时，指定成员变量的值。

  • 由于该注解的保留策略是RetentionPolicy.RUNTIME，故可在运行期通过反射机制来使用，注解实现的就是元数据的第二个作用：代码分析。
